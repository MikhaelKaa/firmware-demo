# firmware-demo
**Multi-MCU demo with hardware abstraction layer**

Демо-проект, демонстрирующий подход к абстракции логики приложения от аппаратной части в встроенных системах на базе ARM Cortex-M.

## Ключевая идея

Проект показывает, как построить **высокоуровневую абстракцию** для логики приложения, независимо от конкретного HAL (STM32 HAL, CMSIS или другой). Приложение работает с устройствами через единый интерфейс, не зная о нижней реализации.

### drv_face — пример для последовательных устройств

Структура `drv_face_t` (read/write/ioctl) — это пример того, как можно унифицировать работу с устройствами последовательной природы: UART, некоторые экраны, клавиатуры и подобные. Она не является центральным паттерном всего проекта, а скорее одним из возможных вариантов унифицированного интерфейса.

## Структура проекта

```
├── firmware/
│   ├── Makefile              # Главный файл сборки (только CLI-модули)
│   ├── local_example.mk      # Пример локальной конфигурации OpenOCD
│   ├── cli/                  # Прикладные модули (CLI команды)
│   │   ├── u_read_line/      # MicroRL + командный процессор
│   │   ├── mem/              # Просмотр памяти
│   │   ├── time/             # Работа со временем
│   │   ├── led/              # Управление LED
│   │   ├── w25q/             # Внешняя флеш W25Q
│   │   └── usb_cdc/          # USB CDC команды
│   ├── core/                 # Submodule: firmware-core (ядро, драйверы, vendor)
│   │   ├── device.mk         # Конфигурация чипа, пути к драйверам, линкер
│   │   ├── kernel/           # Ядро: точка входа, системные вызовы
│   │   ├── drivers/          # Драйверы периферии
│   │   │   ├── adc/
│   │   │   ├── led/          # PWM LED
│   │   │   ├── mem/
│   │   │   ├── rcc/          # Тактирование
│   │   │   ├── rng/          # Генератор случайных чисел
│   │   │   ├── rtc/          # Реальное время
│   │   │   ├── uart/         # UART1, UART2
│   │   │   ├── usb_cdc/      # USB CDC
│   │   │   └── w25q/         # W25Q внешняя флеш
│   │   ├── lib/              # Платформонезависимые библиотеки
│   │   │   └── time/         # Точное время (precise_time.h), микросекундные задержки
│   │   └── vendor/           # CMSIS, startup, linker, OpenOCD конфиги
│   └── build/                # Результат сборки
└── .gitmodules               # Submodule конфигурация
```

### Слои проекта

- **`firmware/Makefile`** — бизнес-логика и CLI-команды. Не знает о железе.
- **`firmware/core/`** (submodule) — драйверы, ядро, vendor-файлы. Содержит `device.mk` со всей конфигурацией сборки для конкретного MCU.

## Сборка

### Требования

- ARM GCC toolchain (`arm-none-eabi-gcc`)
- OpenOCD (для загрузки)
- Make

### Команды

```bash
# Собрать проект
cd firmware
make

# Загрузить в микроконтроллер
make load

# Очистить сборку
make clean

# Настроить VS Code для отладки
make setup-vscode
```

Результаты сборки:
- `build/firmware.elf` — ELF бинарник
- `build/firmware.hex` — Intel HEX
- `build/firmware.bin` — сырой бинарный файл

## Поддерживаемое оборудование

- **Микроконтроллер**: STM32F407VG
- **Программаторы**: ST-Link, CMSIS-DAP (через OpenOCD)
- **Интерфейс отладки**: USB CDC (виртуальный COM-порт)

## CLI команды

После загрузки устройство появляется как виртуальный COM-порт. Доступен командный интерфейс с поддержкой:
- Помощь (`help`)
- Управление LED
- Работа с RTC
- Просмотр памяти
- Работа с внешней флеш W25Q
